# 30 DAY LAUNCH SYSTEM

## Objective
Get TITANIUS_CERBERRUS from empty public repo to first paying customer within 30 days.

## Rule
No decorative complexity until someone pays.

## Week 1: Shape the offer
Deliverables:
- define one niche
- define one painful workflow
- define one promise
- prepare sample input and sample output

Recommended niche:
- local service businesses or solo consultants

Recommended promise:
- recover ignored leads and identify the highest-value follow-ups in one report

Repo tasks:
- create `core/`, `agents/`, `workflows/`, `config/`, `reports/`, `tests/`
- add CLI entrypoint
- add demo dataset
- add dry-run and smoke-test commands

## Week 2: Build the sellable demo
Deliverables:
- one functioning lead recovery workflow
- one markdown report generator
- one JSON result envelope
- one short demo video or GIF

Required user-facing outputs:
- ranked recovery queue
- estimated opportunity value
- suggested next steps
- optional message templates

## Week 3: Sell manually
Deliverables:
- outreach script
- one-page landing copy
- payment link
- onboarding form

Manual sales motion:
1. identify 20 prospects
2. send short direct outreach
3. offer a low-friction audit
4. deliver manually if needed
5. convert best responses into setup service

## Week 4: Productize
Deliverables:
- improve configs from real client feedback
- create a recurring subscription version
- standardize reporting
- document onboarding and operator workflow

## Offer stack

### Entry
Revenue Leak Audit
- 99 to 249 EUR
- turnaround: 24 to 72 hours

### Core
Lead Recovery Setup
- 300 to 900 EUR
- includes configuration and reporting

### Recurring
Weekly Recovery Engine
- 79 to 299 EUR per month
- recurring report and opportunity queue

## Non-negotiable KPIs
- first 20 prospects contacted
- first 5 calls or serious replies
- first paid audit
- first paid setup
- first recurring customer

## What to avoid
- building a giant framework before one workflow sells
- talking about agents more than outcomes
- waiting for perfect branding
- adding dashboards before the report itself is useful

## Simple operating loop
1. ingest messy lead data
2. score and rank opportunities
3. generate report
4. deliver report
5. collect outcome data
6. improve scoring
7. turn manual steps into modules

## Exit condition for this phase
This phase is complete when the repo produces one workflow that a real customer will pay for, even if some fulfillment is still manual behind the scenes.
