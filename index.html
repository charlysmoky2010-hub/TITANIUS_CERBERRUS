Public storefront for practical revenue workflows

Turn messy prospect data into a ranked revenue recovery queue.

TITANIUS_CERBERRUS helps founders, creators, consultants, agencies, and small teams recover missed opportunities by scoring stale leads, prioritizing follow-ups, and turning pipeline chaos into usable action.

The first public workflow is lead_recovery_audit. It reads a lead list, normalizes statuses, excludes dead records, scores recovery potential, and outputs a queue you can actually act on.

View GitHub Repo Read README Example Output

What it does

  • Ingests JSON or CSV lead data
  • Normalizes messy pipeline states
  • Excludes won, lost, and uncontactable records
  • Scores remaining leads by recovery potential
  • Generates JSON and Markdown outputs

Who it is for

  • Solo founders
  • Creators with inbound demand
  • Consultants and service businesses
  • Agencies with stale pipelines
  • Small teams that need a lightweight recovery workflow before a heavy CRM stack

How to run it locally

Why this repository is narrow on purpose

This is not a generic agent demo pretending to do everything. The public repository is intentionally focused on one practical workflow that sits close to revenue: deciding which cold or neglected leads are still worth following up, and what to do next.
